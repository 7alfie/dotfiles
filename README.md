# ✦ Hyprland Dotfiles

> A sleek, developer-focused Hyprland setup for Arch Linux  
> **Fully dynamic theming** via [Matugen](https://github.com/InioX/matugen) — your entire colour scheme regenerates from your wallpaper automatically.

---

| Role | Tool |
|------|------|
| Compositor | [Hyprland](https://hyprland.org/) |

| Keyboard Layout | **GB (United Kingdom)** |

## How Dynamic Theming Works

```
  Wallpaper selected
        │
        ▼
  swww sets the wallpaper
        │
        ▼
  matugen reads the image
  → derives a Material You palette
  → renders templates for:
        ├── hypr/colors.conf     (border colours)
        ├── waybar/colors.css    (bar palette)
        ├── wofi/colors.css      (launcher palette)
        ├── mako/config          (notification colours)
        └── kitty/colors.conf    (terminal colours)
        │
        ▼
  Waybar & Mako reload
  Hyprland config reloads
  → Everything matches your wallpaper instantly
```

Press `SUPER + SHIFT + W` at any time to cycle to a new wallpaper and retheme.

## Installation

### 1 — Install packages

```bash
sudo pacman -S --needed \
  hyprland waybar wofi mako libnotify swww \
  kitty neovim git curl wget \
  thunar firefox \
  grim slurp \
  xdg-desktop-portal-hyprland xdg-utils \
  polkit-kde-agent qt5-wayland qt6-wayland \
  noto-fonts noto-fonts-emoji ttf-jetbrains-mono-nerd \
  ripgrep fd lazygit btop fastfetch \
  python-pillow       # required by matugen
```

Install AUR packages (requires `yay`):

```bash
# Install yay first if needed
git clone https://aur.archlinux.org/yay.git /tmp/yay
cd /tmp/yay && makepkg -si

# AUR packages
yay -S grimblast-git matugen-bin
```

### 2 — Clone and install dotfiles

```bash
git clone https://github.com/YOUR_USERNAME/dotfiles.git ~/dotfiles
cd ~/dotfiles
chmod +x install.sh
./install.sh
```

### 3 — Add wallpapers

```bash
# Drop .jpg / .png / .webp files into:
~/Pictures/wallpapers/
```

Good sources: [Wallhaven](https://wallhaven.cc) · [Unsplash](https://unsplash.com)

### 4 — Install LazyVim

```bash
# Backup existing config if any
mv ~/.config/nvim ~/.config/nvim.bak 2>/dev/null

# Clone LazyVim starter
git clone https://github.com/LazyVim/starter ~/.config/nvim
rm -rf ~/.config/nvim/.git

# Launch — it auto-installs everything on first run
nvim
```

On first launch press `q` to dismiss prompts and wait for plugins to install. Restart nvim once done.

**Essential LazyVim keys:**
- `<Space>` — discover all keybinds (which-key)
- `<Space>ff` — find files
- `<Space>fg` — live grep across project
- `<Space>e` — file tree (Neo-tree)
- `<Space>gg` — Lazygit TUI

### 5 — Log in

Reboot. At the **Ly** login manager, select **Hyprland**. On login, Hyprland will automatically start Waybar, Mako, swww, and set your first wallpaper + generate the initial theme.

## File Structure

```

│   ├── hyprland.conf          # Main compositor config + keybinds

│   ├── style.css              # Launcher styling (imports colors.css)
│   └── colors.css             # Auto-generated by matugen
├── mako/
│   └── config                 # Auto-generated by matugen
├── kitty/
│   ├── kitty.conf             # Terminal config (includes colors.conf)
│   └── colors.conf            # Auto-generated by matugen

│   ├── config.toml            # Template paths and settings

│       ├── hyprland-colors.conf

│       ├── wofi-colors.css
│       ├── mako
│       └── kitty-colors.conf
└── scripts/
    └── wallpaper.sh           # Wallpaper picker + matugen trigger
```

## Troubleshooting

**Waybar shows errors:**
```bash
waybar 2>&1   # run manually to see the error output
```

**Matugen not found:**
```bash
yay -S matugen-bin
```

**No theme generated yet (borders look wrong):**
```bash
~/.config/scripts/wallpaper.sh   # run manually
```

**Screen tearing or black screen:**
```
# Add to hyprland.conf:
env = WLR_NO_HARDWARE_CURSORS,1
```

**Wofi style not applying:**
```bash
# Make sure colors.css exists — it won't until matugen runs once
~/.config/scripts/wallpaper.sh
```
